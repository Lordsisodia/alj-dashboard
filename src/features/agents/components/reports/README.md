# Reports Pill — ISSO Agents Dashboard

Reports is one of three tabs in the ISSO Agents dashboard (`/isso/agents`). It surfaces AI-generated intelligence reports from agent runs — scrapers, schedulers, and analyst agents write report records that appear here in a visually rich card layout.

## Overview

### What Is the Reports Pill?

The Reports pill displays a collection of AI-generated reports extracted from agent activity. Each report contains:
- A **title** (e.g., "Weekly Intelligence Report — Apr 6, 2026")
- A **category** (Intelligence, Recon, or Performance)
- **Insights** — numbered findings from the agent analysis
- **Generated by** — which agent created the report
- **Timestamp** — when the report was generated

### Use Case

OFM (OnlyFans Management) agencies use this to:
- Review automated competitive intelligence (Recon reports)
- Check performance metrics and trends (Performance reports)
- Monitor AI-driven insights from their creator content (Intelligence reports)
- Make data-driven decisions about content strategy

## Component Architecture

```
src/features/agents/components/reports/
├── ReportsView.tsx         # Main exported view — stats bar + filter chips + content
├── ReportCard.tsx          # Individual report card — collapsed & expanded states
├── index.ts                # Public exports
└── utils.ts                # Shared relativeTime() formatter
```

### Component Responsibilities

| Component | Purpose |
|-----------|---------|
| **ReportsView.tsx** | Entry point; renders stats bar, category filters, and report list. Manages active filter state. Contains error boundary and live/static content switching. |
| **ReportCard.tsx** | Individual report card with collapsed card state, expanded accordion insights, animations, and interactive buttons. |
| **utils.ts** | Shared `relativeTime()` utility to convert unix millisecond timestamps to human-readable strings ("2h ago", "3d ago"). |
| **index.ts** | Public exports: `ReportsView`, `ReportCard`. |

### Data Types

Types are defined in `src/features/agents/types.ts`:

```typescript
export type ReportCategory = 'Intelligence' | 'Recon' | 'Performance';

export interface Report {
  id: string;
  title: string;
  insights: string[];
  generatedBy: string;
  generatedAt: number;          // Unix milliseconds
  category: ReportCategory;
}
```

## Features

### 1. Report Cards — Collapsed State

Each report renders as a white card with:

- **3px gradient top accent bar** — colored per category:
  - Intelligence: `#ff0069` (pink)
  - Recon: `#4a9eff` (blue)
  - Performance: `#78c257` (green)

- **Category badge** — pill badge with category color background (uppercase, tracked)

- **Generated By badge** — Bot icon + agent name, tinted in category color

- **NEW badge** — solid category color pill with Sparkles icon, shown if report is < 24 hours old

- **Insight count chip** — "N insights" text in neutral gray

- **Bold title** — primary text describing the report

- **Preview insight** — first insight in a left-bordered tinted box for context

- **Relative timestamp** — "2h ago", "3d ago", computed client-side

- **Read Full Report button** — toggles expanded state; fills solid category color when active, ChevronDown rotates 180°

- **Hover lift** — when collapsed, card lifts (`y: -2`) with subtle shadow on hover

Example markup (simplified):

```tsx
<motion.div className="rounded-xl">
  <div style={{ height: 3, background: `linear-gradient(90deg, ${catColor}, ${catColor}88)` }} />
  <div className="p-5 space-y-3.5">
    {/* category + generated-by + NEW badge */}
    <div className="flex items-center justify-between gap-2">
      {/* badges */}
    </div>
    {/* title */}
    <h3 className="text-sm font-bold">{report.title}</h3>
    {/* preview insight */}
    <div style={{ backgroundColor: `${catColor}08`, borderLeft: `2px solid ${catColor}50` }}>
      {previewInsight}
    </div>
    {/* footer + read full report button */}
    <div className="flex items-center justify-between pt-1">
      <span className="text-[11px] text-neutral-400">{relativeTime(report.generatedAt)}</span>
      <button onClick={() => setExpanded(!expanded)}>
        {expanded ? 'Collapse' : 'Read Full Report'}
        <ChevronDown />
      </button>
    </div>
  </div>
</motion.div>
```

### 2. Expanded Accordion — Full Insights

When "Read Full Report" is clicked, the card expands with animation:

- **Animated height transition** — 0 → auto over 280ms with custom easing (`[0.25, 0.1, 0.25, 1]`)

- **Tinted background** — 6% opacity of category color (`${catColor}06`)

- **"ALL INSIGHTS" section label** — uppercase, tracked, in category color

- **Numbered insight list** — each insight rendered as:
  ```
  [1] First insight text
  [2] Second insight text
  ...
  ```
  - Numbers in small colored circles with category color background
  - Text in neutral gray

- **Footer section**:
  - **Left**: generated-by agent name + relative timestamp, tinted in category color
  - **Right**: solid category color "Export" button (UI-only, no real download yet)

### 3. Category Filter Chips

Four filter buttons above the report list:

| Chip | Behavior |
|------|----------|
| **All** | Active: dark gray fill. Inactive: transparent with subtle border. Shows all reports. |
| **Intelligence** | Active: pink fill (`#ff0069`). Inactive: transparent. Filters to Intelligence reports only. |
| **Recon** | Active: blue fill (`#4a9eff`). Inactive: transparent. Filters to Recon reports only. |
| **Performance** | Active: green fill (`#78c257`). Inactive: transparent. Filters to Performance reports only. |

Filter state is managed in `ReportsView` via `activeCategory` state. Filtering is client-side.

```tsx
const filtered = activeCategory === 'All'
  ? allReports
  : allReports.filter(r => r.category === activeCategory);
```

### 6. Error Boundary & Static Fallback

The `ReportsView` component wraps live Convex content in an error boundary:

```tsx
<ReportsErrorBoundary fallback={<StaticReportsContent category={activeCategory} />}>
  <LiveReportsContent category={activeCategory} />
</ReportsErrorBoundary>
```

If Convex backend throws (e.g., "function not found"), the error boundary catches it and falls back to `StaticReportsContent`, which renders using the `REPORTS` constant from `src/features/agents/constants.ts`. This ensures the Reports pill always shows data, even if the Convex backend hasn't deployed yet.

## Data Flow

### Live Data Path (Convex)

```
Convex DB (agentReports table)
  ↓
useQuery(api.agents.getAgentReports)  [hooks/index.ts → useReports()]
  ↓
LiveReportsContent
  ├── Loading → ReportsSkeleton
  ├── Loaded  → ReportList → ReportCard[]
  └── Error   → ErrorBoundary → StaticReportsContent → ReportCard[]
```

### Convex Backend Integration

The hook `useReports()` (in `src/features/agents/hooks/index.ts`) calls:

```typescript
const raw = useQuery(api.agents.getAgentReports);

return raw
  .slice()
  .sort((a, b) => b.generatedAt - a.generatedAt)  // descending by timestamp
  .map(r => ({
    id: r._id as string,
    title: r.title,
    insights: r.insights,
    generatedBy: r.generatedBy,
    generatedAt: r.generatedAt,
    category: r.category as ReportCategory,
  }));
```

**Convex Function Expected Signature:**

```typescript
// convex/agents.ts
export const getAgentReports = query(async (ctx) => {
  return await ctx.db
    .query('agentReports')
    .order('desc')  // optional: Convex handles sorting
    .collect();
});
```

**Convex Schema (convex/schema.ts):**

```typescript
agentReports: defineTable({
  title: v.string(),
  category: v.union(
    v.literal('Intelligence'),
    v.literal('Recon'),
    v.literal('Performance')
  ),
  insights: v.array(v.string()),
  generatedBy: v.string(),
  generatedAt: v.number(),  // Unix ms timestamp
}).index('by_generated_at', ['generatedAt'])
  .index('by_category', ['category'])
```

## Styling & Colors

### Color System

All colors are defined in `src/features/agents/constants.ts`:

```typescript
export const REPORT_CAT_COLOR: Record<ReportCategory, string> = {
  Intelligence: '#ff0069',  // Pink
  Recon:        '#4a9eff',  // Blue
  Performance:  '#78c257',  // Green
};
```

Every report card derives its accent color from `REPORT_CAT_COLOR[report.category]`.

### Opacity Map

The same category color is applied at varying opacities for visual hierarchy:

| Opacity | Use Case |
|---------|----------|
| 100% (solid) | Active buttons, expanded state accent bar, NEW badge, Export button |
| 20% | Insight number circle background |
| 18% | Category badge background, expanded section border-top |
| 15% | Footer divider in expanded section |
| 12% | Inactive button border, category badge background |
| 10% | Generated-by badge background |
| 8% | Preview insight box background |
| 6% | Expanded section background |

### Tailwind + Inline Styles

The component uses a mix of Tailwind and inline styles:

- **Tailwind**: spacing (`p-5`, `gap-2`), sizing (`text-sm`), layout (`flex`, `rounded-xl`)
- **Inline styles**: dynamic colors, gradients, opacity-driven backgrounds (`${catColor}08`)

This hybrid approach allows for category-specific theming without hardcoding colors.

### Animations

- **Card load**: `fadeUp` variant — slide in from below with fade
  ```typescript
  const fadeUp = {
    hidden: { opacity: 0, y: 12 },
    visible: { opacity: 1, y: 0, transition: { duration: 0.35, ease: [0.25, 0.1, 0.25, 1] } },
  };
  ```

- **Card hover**: Lift up slightly when not expanded
  ```tsx
  whileHover={expanded ? undefined : { y: -2, boxShadow: '0 6px 20px rgba(0,0,0,0.09)', transition: { duration: 0.18 } }}
  ```

- **Expanded accordion**: Height transition from 0 → auto
  ```tsx
  animate={{ height: 'auto', opacity: 1 }}
  transition={{ duration: 0.28, ease: [0.25, 0.1, 0.25, 1] }}
  ```

- **Chevron rotation**: 180° when expanded
  ```tsx
  animate={{ rotate: expanded ? 180 : 0 }}
  transition={{ duration: 0.2 }}
  ```

- **Container stagger**: Reports fade in with staggered timing
  ```typescript
  const containerVariants = {
    hidden: {},
    visible: { transition: { staggerChildren: 0.06 } },
  };
  ```

## Troubleshooting

### No reports appear

1. Check Convex function `api.agents.getAgentReports` is deployed
2. Verify `agentReports` table exists in Convex schema
3. Confirm the error boundary is not suppressing an error (check browser console)
4. Ensure `REPORTS` constant is populated in `src/features/agents/constants.ts`

### Colors are wrong

1. Verify `REPORT_CAT_COLOR` is defined correctly in `constants.ts`
2. Ensure report's `category` field matches one of `'Intelligence' | 'Recon' | 'Performance'`

### Animations not working

1. Confirm Framer Motion is installed: `npm list framer-motion`
2. Check that `motion`, `AnimatePresence` are imported from `framer-motion`
3. Verify Tailwind's `animate-pulse` class is available (requires Tailwind CSS)
