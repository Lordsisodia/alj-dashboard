# Story 02.12: Mock Data Refinement & Feature Flags

<!-- Powered by BMAD™ Core -->

## Status
Backlog

## Story
**As a** front-end developer working with placeholders,
**I want** mock data kept lightweight and flag-able,
**so that** performance stays predictable even while we demo unfinished features.

## Acceptance Criteria

1. **Data Weight Budget**
   - Every mock data module under `src/domains/partnerships/**/data` stays ≤ 10 KB parsed per import.
   - Large showcase datasets (portfolio, community, earnings) gated behind feature flags or split into chunks.

2. **Feature Flag System**
   - Introduce lightweight flag helper (env-based) to disable heavy mock sections (e.g., “show 4 client case studies vs 18”).
   - Document default flag values for perf testing vs marketing demos.

3. **Audit + Cleanup**
   - Run analyzer to list modules over budget; trim or split according to guidelines.
   - Add script to verify budgets (`npm run perf:mock-budget`).

4. **Docs & Communication**
   - Update plan/checklist with mock data policies + flag table.
   - Provide quick reference for marketers toggling demo content without code changes.

## Tasks / Subtasks

- [ ] **Mock Data Audit** (AC: 1,3)
  - [ ] Script to parse bundle analyzer JSON and output modules >10 KB.
  - [ ] Refactor offending modules (split, move to JSON, lazy-load).

- [ ] **Feature Flag Helper** (AC: 2)

  - [ ] Use flags in portfolio, community, earnings to limit mock payload.

- [ ] **Budget Check Script** (AC: 3)

- [ ] **Docs** (AC: 4)
