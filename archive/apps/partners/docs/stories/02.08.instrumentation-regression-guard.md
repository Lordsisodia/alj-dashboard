# Story 02.08: Perf Instrumentation & Regression Guard

<!-- Powered by BMAD™ Core -->

## Status
Backlog

## Story
**As a** performance-minded partner team,
**I want** automated guardrails that re-run the 88-route baseline and enforce budgets in CI,
**so that** future UI work can’t silently regress our sub‑2 s targets.

## Acceptance Criteria

1. **Automated Baseline Command**
   - `npm run perf:baseline` supports `BASELINE_DIR`, `ROUTE_TIMEOUT_MS`, `POST_LOAD_DELAY_MS`, and `PORT` overrides documented in README.
   - Command can target both dev (`next dev`) and prod (`next start`) servers via a single flag.

2. **CI Integration**
   - GitHub Actions workflow runs `npm run perf:baseline` (prod build) weekly + on demand, publishes TSV diff artifacts.
   - Workflow fails if any `/partners/*` route exceeds cluster budgets listed in `docs/partners/perf/bmad-performance-plan.md`.

3. **TSV Diff & Alerting**
   - New script `scripts/perf/diff-baselines.mjs` compares latest TSV to a reference and prints delta table.
   - When budgets exceeded, CI comment (or log) lists offending routes + deltas.

4. **Documentation**
   - `docs/partners/perf/bmad-performance-plan.md` + checklist updated with instructions for running dev vs prod baselines.
   - README snippet added under `docs/partners/perf/` explaining how to interpret TSV + trace artifacts.
