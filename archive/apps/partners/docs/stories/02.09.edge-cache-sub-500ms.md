# Story 02.09: Edge Cache & Sub-500 ms TTI

## Status
Backlog

## Story
**As a** performance-obsessed partner team,
**I want** the production build to hit <500 ms TTI for benchmark routes via edge caching and optimized media,
**so that** partners see instant experiences even on cold loads.

## Acceptance Criteria

1. **Prod Baseline**
   - Run `npm run build && next start` with `npm run perf:baseline` (prod mode) and store artifacts under `docs/partners/perf/baselines/2025-11-XX-prod/`.
   - `/partners/academy`, `/partners/pipeline-ops`, `/partners/recruitment` show load ≤ 600 ms / FCP ≤ 500 ms when cached.

2. **Edge Cache Policy**
   - Introduce middleware/headers to cache static `/partners` pages at the edge (`s-maxage`, `stale-while-revalidate`).
   - Document invalidation strategy and confirm via devtools (response headers) + repeated requests.

3. **Critical Media Optimization**
   - Each benchmark route has exactly one `fetchpriority="high"` asset; all other hero media load lazily with aspect-ratio placeholders.
   - LCP images sized responsively and preloaded via `<link rel="preload" as="image">` where needed.

4. **CI Budget**
   - Add script `npm run perf:tti` that reads prod baseline TSV and fails if TTI > 500 ms for benchmark routes.
   - CI job runs this script weekly.
