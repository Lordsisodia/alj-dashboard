# Story 02.07: Legal, Privacy & Misc Simplification

<!-- Powered by BMAD™ Core -->

## Status
Backlog

## Story
**As a** partner accessing legal/terms/privacy content,
**I want** those static pages to render instantly,
**so that** docs feel professional and don’t waste shell bandwidth.

## Acceptance Criteria

1. **Performance Budget**
   - `/partners/privacy-policy`, `/partners/settings/legal/*`, `/partners/settings/privacy`, `/partners/settings/security`, `/partners/settings/language`, `/partners/settings/integrations` load ≤ 1,000 ms / FCP ≤ 800 ms (baseline 0.6–3.0 s per TSV rows 55–81).

2. **Pure Server Components**
   - All legal/privacy routes drop unnecessary `"use client"` imports; they render as RSC-only with zero client JS except shared shell.
   - Backgrounds swap to static gradients (no Waves) to avoid hydration.

3. **Documentation & Accessibility**
   - Provide accessible skip links + headings (WCAG 2.1) for legal content.
   - Add lint rule/snippet preventing future client imports in these routes.

4. **Measurement**
   - Perf sweep verifies budgets; analyzer shows each legal route ≤ 40 KB parsed.
   - Plan/checklist updated; story notes include guidance for future contributors.
