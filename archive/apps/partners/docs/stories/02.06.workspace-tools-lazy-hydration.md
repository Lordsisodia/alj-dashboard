## Story
**As a** partner editing plans and workspace assets,
**I want** the tools/notes/files pages to feel instant even with editor stubs,
**so that** Pro partners can demo the UI without 2–5 s delays.

## Acceptance Criteria

1. **Performance Budget**
   - `/partners/tools/app-plan-generator`, `/partners/workspace`, `/partners/workspace/files`, `/partners/workspace/tasks`, `/partners/workspace/notes/my-notes`, `/partners/workspace/calendar` load ≤ 1,500 ms / FCP ≤ 1,200 ms (baseline 0.9–5.6 s in TSV rows 82–88).

2. **Editor Lazy Loading**
   - Monaco/editor-like dependencies, AI agent prompts, and large code snippets wrapped in `next/dynamic` with `ssr:false` and hydrate on interaction.
   - Read-only cards render server-side with placeholders until editors load.

3. **Data Streaming**
   - Workspace list/grid data streamed server-side from JSON or mock API; no inline arrays inside client files.
   - Calendar/tasks views fetch data via RSC to avoid bundling giant objects client-side.

4. **Measurement**
   - Perf sweep proves all workspace/tools routes ≤ 1.5 s load; results logged + checklist updated.
   - Analyzer output shows bundle per route ≤ 90 KB parsed.
