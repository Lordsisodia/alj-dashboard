# Story 02.05: Shell & Mobile Core Slimdown

<!-- Powered by BMAD™ Core -->

## Status
Backlog

## Story
**As a** partner bouncing between tabs,
**I want** the Campus shell + mobile nav to hydrate instantly,
**so that** every route inherits a <1 s shell overhead instead of 2–4 s baseline costs.

## Acceptance Criteria

1. **Performance Budget**
   - Shell-dependent routes (`/partners`, `/partners/campus`, `/partners/checklist`, `/partners/messages`, `/partners/profile`, `/partners/settings/*`) add ≤ 500 ms overhead (load ≤ 2,000 ms for `/partners`, ≤ 1,500 ms for the rest).
   - Baseline reference: `/partners` load 4,375 ms; `/partners/messages` 775 ms; `/partners/settings/general` 451 ms (needs to stay ≤ 500 ms even after refactor).

2. **Lightweight Nav Data**
   - Campus navigation data (menu items, quick actions) fetched via JSON + `cache(fetch)` (≤ 10 KB) instead of bundling static arrays (~57 KB parsed today).
   - Mobile nav state persisted via `sessionStorage` or server hints so re-navigation doesn’t rehydrate entire store.

3. **Animation & Wave Gating**
   - `Waves` background only loads on hero pages; other routes use CSS gradients.
   - Floating nav button + drawer hydrate only on first interaction.

4. **Instrumentation**
   - Perf sweep shows `/partners` load ≤ 2 s, `/partners/campus` ≤ 1.5 s; results logged.
   - Checklist updated; doc notes include instructions for keeping nav data externalized for future features.
