# Story 02.10: Perf Telemetry & Sentry Dashboard

## Status
Backlog

## Story
**As a** partner engineering lead,
**I want** live performance telemetry (web-vitals + custom marks) flowing into Sentry/New Relic dashboards,
**so that** regressions show up immediately after deploy—not just during manual sweeps.

## Acceptance Criteria

1. **Client Telemetry**
   - `PartnerPerfMetrics` sends FCP/LCP/CLS/INP plus custom marks (hydration_end, first_interaction) to Sentry (or chosen APM) in production builds.
   - Sampling configurable via env var; default 100% in staging, 10% in prod.

2. **Route Tagging**
   - Each event includes route pattern (e.g., `/partners/community/[slug]`), cluster tag, and build commit SHA.
   - Telemetry captures nav type (SSR vs client transition).

3. **Dashboards & Alerts**
   - Create Sentry dashboard (or NR equivalent) tracking median + p90 FCP/LCP per cluster.
   - Alert triggers when cluster budgets exceeded for two consecutive deploys.

4. **Docs & Ops**
   - Update perf plan/checklist with telemetry steps (how to enable, sample rates, dashboards).
   - Add runbook snippet describing how to investigate + link to `npm run perf:baseline` when alerts fire.

## Tasks / Subtasks

- [ ] **Telemetry Hook** (AC: 1,2)
  - [ ] Extend `PartnerPerfMetrics` to send events to Sentry using `@sentry/nextjs` (or telemetry endpoint) with route tags.
  - [ ] Add config for sample rate + environment gating.

- [ ] **Route Pattern Tagging** (AC: 2)
  - [ ] Create helper that maps `window.location.pathname` to route pattern (use manifest from `scripts/perf/capture-partner-traces.mjs`).
  - [ ] Attach cluster tag (shell, academy, etc.).

- [ ] **Dashboard & Alerts** (AC: 3)
  - [ ] Build Sentry dashboard (FCP/LCP median/p90 per cluster) + share link.

- [ ] **Docs** (AC: 4)
  - [ ] Update plan/checklist + README with telemetry instructions and runbook.
