# Story 02.11: Adaptive Hydration Lint & Component Library

<!-- Powered by BMAD™ Core -->

## Status
Backlog

## Story
**As a** front-end contributor,
**I want** lint rules and ready-made components for skeletons/dynamic hydrators,
**so that** new pages automatically follow our streaming + lazy-hydration standards without regressions.

## Acceptance Criteria

1. **Lint Rules / Codemods**
   - ESLint rule (or codemod) fails if a `/partners` route imports large mock data inside client components or forgets to wrap heavy widgets in `next/dynamic`.
   - Rule checks for `"use client"` misuse (server page importing it without need) and suggests server components.

2. **Skeleton & Hydrator Library**
   - `@/components/perf/SuspenseSkeleton.tsx`, `HydrateOnView.tsx`, etc., exported as documented utilities.
   - Guidelines cover skeleton sizing, fallback durations, and `IntersectionObserver` hook.

3. **Developer Guide**
   - Short doc (e.g., `docs/partners/perf/hydration-handbook.md`) showing patterns, lint errors, and how to fix them.
   - Linked from plan/checklist + README.

4. **Verification**
   - Run lint on existing codebase; open issues (or fix) for each violation.
   - Add CI step `npm run perf:lint` to enforce rules moving forward.

## Tasks / Subtasks

- [ ] **Rule Implementation** (AC: 1)
  - [ ] Extend ESLint config with custom rule(s) or write codemod to detect large inline arrays/client imports.
  - [ ] Provide autofix suggestions where possible.

- [ ] **Skeleton/Hydrator Components** (AC: 2)
  - [ ] Create shared skeleton + hydration components with TS docs.
  - [ ] Replace ad-hoc implementations in at least one route per cluster to validate usage.

- [ ] **Documentation** (AC: 3)
  - [ ] Write hydration handbook referencing plan + examples.
  - [ ] Update plan/checklist references.

- [ ] **CI Hook** (AC: 4)
  - [ ] Add `npm run perf:lint` script + CI step.
  - [ ] Document failure remediation.
